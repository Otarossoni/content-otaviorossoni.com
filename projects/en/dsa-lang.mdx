Since I started studying programming, I realized that understanding algorithms and data structures is fundamental for any developer. It's not just about passing technical interviews, but about developing a logical mindset that applies to any problem. It was with this mentality that I created DSA.lang — a personal catalog of everything I've been studying.

## The Project

DSA.lang is a repository that catalogs each algorithm or data structure learned, accompanied by notes on how it works and the logic applied. The idea is that each entry contains not just the code, but also a clear explanation of how and why it works.

The project is organized into three main areas:

- **Algorithms** — implementations of sorting, searching, and other algorithms;
- **Data Structures** — implementations of linear and non-linear structures;
- **Exercises** — solutions to problems from platforms like LeetCode, HackerRank, and similar.

## Repository Structure

Each algorithm or data structure follows a standard organization: a dedicated folder containing implementations in multiple languages and a Markdown file with explanations. This allows comparing how the same concept behaves across different ecosystems.

## Goal

More than a code repository, DSA.lang is a study tool. Each new concept learned is documented and implemented, serving as a future reference. It's a project in constant evolution, growing alongside my learning.
