In the second half of 2023, I had the opportunity to conduct a workshop at UPF (Universidade de Passo Fundo) during the OpenTI II event. The chosen topic was something I had been studying and applying daily: Clean Code best practices with NodeJS.

The workshop proposal was simple yet ambitious: in a single session, demonstrate the importance of Clean Code in software development and, in practice, build code examples applying these concepts. The idea was that participants would leave not just with functional code snippets, but with a different mindset about how to write software.

## Workshop Structure

The workshop was divided into three progressive modules:

- **Introduction** — contextualization about what Clean Code is and why it matters;
- **Best Practices** — exploration of principles and conventions that make code more readable, maintainable, and scalable;
- **Application** — hands-on development of code examples with TypeScript and NodeJS, applying all the concepts discussed.

The focus was not on specific frameworks or tools, but on fundamentals that transcend any technology: meaningful names, small functions, single responsibility, and code that explains itself.

## Technologies Used

For the practical part, I used:

- **Node.js**: JavaScript runtime environment on the server;
- **TypeScript**: the workshop's main language, used to add static typing and improve code quality.

The choice of TypeScript was deliberate — I wanted to show that Clean Code is not about the language, but about how you use it. TypeScript, with its type system, offers an extra layer of clarity that reinforces Clean Code principles.
