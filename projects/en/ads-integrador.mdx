From the beginning of the Systems Analysis and Development course, we had already been informed that the Final Project would have a much more practical than theoretical weight — although the latter would not be set aside. Still, during the first semesters, while learning programming, I had no idea what I would present at the end of the degree.

As time passed and deadlines approached, my initial idea was to develop a service aggregator system for the city hall. I managed to implement a considerable part of the solution, but when only three months were left for the final presentation, I realized I wasn't really satisfied with the project. That's when I made a decision that, for many, seemed reckless: start everything from scratch.

My advisor was openly against the idea. He presented me with several reasons to give up, the deadline being the main one. But I was convinced I could deliver something better. So, I abandoned the system that was already in development and started over.

I didn't want to create just another project with a generic name and a dozen CRUDs. I wanted to develop something I myself would want to use, that had personality and even a bit of humor. That's how **PromoCão — The Deal Sniffer** was born.

In the following three months, I dedicated practically all my free time to the project. As it evolved, I added details, refinements, and small subtleties that helped build its identity.

In the end, in addition to drawing some smiles from the veterans and the evaluation committee, the project earned me the highest grade in the presentation. More important than that, it was the first time I felt I had created software with its own personality, and not just another academic work.

## Features

Among the main features implemented are:

- Registration and authentication of regular users and administrators;
- Promotion submission by the community;
- Promotion approval and moderation workflow;
- Public viewing of approved promotions;
- Administrative area for platform content management.

## Technologies Used

At the time, I chose to use the MERN stack, one of the most popular combinations for full stack JavaScript applications:

- **MongoDB**: NoSQL database used for information storage;
- **Express.js**: framework responsible for building the backend API;
- **React.js**: library used for web interface development;
- **Node.js**: runtime environment that allowed using JavaScript in both frontend and backend.
